# Configuration Reference

Preboot reads one or two YAML files to define which checks to run.

| File | Purpose | Edit manually? |
|---|---|---|
| `preboot-auto.yml` | Auto-generated by `preboot init` | No — regenerate instead |
| `preboot.yml` | User-defined overrides and additions | Yes |

Both files must live in the **current working directory** when you invoke `preboot`.

---

## Full schema

```yaml
# Required. Must be exactly 1.
version: 1

# Optional global defaults applied to every check.
defaults:
  # When true (the default), "warning" severity is treated like "blocker".
  # Set to false to let warnings be informational-only.
  strict: true

  # Milliseconds before a check times out.
  # Applied to every check that doesn't set its own timeout_ms option.
  timeout_ms: 3000

# List of checks to run, in declaration order.
checks:
  - name: string          # Required. Unique human-readable identifier.
    type: string          # Required. One of the 7 registered check types.
    severity: string      # Required. "info" | "warning" | "blocker"
    options:              # Type-specific key/value pairs (see Check Types).
      key: value
    message: string       # Optional. Custom failure message.
    fix: string           # Optional. Remediation instructions shown on failure.
```

---

## Field reference

### `version`

```yaml
version: 1
```

Must be the integer `1`. Validation fails for any other value. Reserved for future format evolution.

---

### `defaults`

```yaml
defaults:
  strict: true
  timeout_ms: 3000
```

Applied to every check before the check runs. A check can override `timeout_ms` via its own `options.timeout_ms`.

**`strict` (bool, default `true`)**

When `true`, checks with `severity: warning` are treated identically to `severity: blocker` — they fail the run and set exit code `1`.

When `false`, warnings are logged with a ⚠️ icon but do not cause a non-zero exit.

**`timeout_ms` (int, default `3000`)**

Default timeout for all checks, in milliseconds. Network checks (`http_reachable`, `tcp_reachable`) respect this. Non-network checks (`command_exists`, `file_exists`, etc.) do not block on I/O but still receive the value for consistency.

---

### `checks[]`

Each entry in the `checks` list represents one diagnostic.

#### `name` (string, required)

Unique, human-readable identifier. Displayed in check output. Used to resolve conflicts during config merging — if `preboot-auto.yml` and `preboot.yml` both declare a check with the same name, the `preboot.yml` version wins.

```yaml
- name: postgres-port-free
```

#### `type` (string, required)

The check implementation to use. Must be one of:

| Value | What it checks |
|---|---|
| `env_exists` | Environment variable present in `.env` |
| `command_exists` | Command available in `$PATH` |
| `file_exists` | File present at a relative path |
| `directory_exists` | Directory present at a relative path |
| `http_reachable` | HTTP/HTTPS endpoint returns 2xx |
| `tcp_reachable` | TCP connection succeeds to host:port |
| `port_free` | TCP port is not currently in use |

Full option documentation for each type: [Check Types](checks.md).

#### `severity` (string, required)

| Value | Meaning |
|---|---|
| `info` | Logged only. Never causes a non-zero exit. |
| `warning` | Causes non-zero exit only when `strict: true` (the default). |
| `blocker` | Always causes non-zero exit. |

#### `options` (map[string]string, optional)

Type-specific configuration. All values must be strings, even numbers:

```yaml
options:
  port: "5432"       # string, not integer
  timeout_ms: "5000"
```

#### `message` (string, optional)

Custom message printed when the check fails, instead of the default error from the check implementation. Use this to give context specific to your project:

```yaml
message: "Postgres is not running — start with 'docker compose up db -d'"
```

#### `fix` (string, optional)

Step-by-step remediation printed below a failed check. Supports multi-line strings:

```yaml
fix: |
  1. Copy .env.example to .env
  2. Open .env and fill in the STRIPE_SECRET_KEY value
  3. Re-run preboot check
```

## Config merging

When neither `--config` flag is used, the engine merges `preboot-auto.yml` and `preboot.yml`:

```
                ┌──────────────┐
                │ preboot-auto.yml │  (generated)
                └──────┬───────┘
                       │
                       │  merge (user overrides auto by name)
                       │
                ┌──────▼───────┐
                │   preboot.yml   │  (hand-written)
                └──────┬───────┘
                       │
                       ▼
               effective config
```

**Rules:**

1. Checks from `preboot-auto.yml` are loaded first.
2. Checks from `preboot.yml` are processed in order:
   - If a check with the same `name` already exists in the auto config → replace it.
   - Otherwise → append it.
3. `defaults` from `preboot.yml` take precedence over `preboot-auto.yml` defaults.

This means you can selectively tighten or relax an auto-generated check by copying its name into `preboot.yml` and adjusting the fields:

```yaml
# preboot.yml — override the auto-generated go-installed check to be a warning
checks:
  - name: go-installed
    type: command_exists
    severity: warning   # was "blocker" in preboot-auto.yml
    options:
      command: go
```

## Complete example

```yaml
version: 1

defaults:
  strict: true
  timeout_ms: 5000

checks:
  # Tooling
  - name: go-installed
    type: command_exists
    severity: blocker
    options:
      command: go
    fix: "Install Go from https://go.dev/dl/"

  - name: make-installed
    type: command_exists
    severity: warning
    options:
      command: make

  # Environment
  - name: env-file-exists
    type: file_exists
    severity: blocker
    options:
      path: .env
    fix: "Run: cp .env.example .env"

  - name: db-url-set
    type: env_exists
    severity: blocker
    options:
      key: DATABASE_URL
    fix: "Set DATABASE_URL in your .env file"

  - name: stripe-key-set
    type: env_exists
    severity: blocker
    options:
      key: STRIPE_SECRET_KEY

  # Services
  - name: postgres-port-free
    type: port_free
    severity: blocker
    options:
      port: "5432"
    message: "Port 5432 is in use — is another Postgres running?"
    fix: "Stop conflicting service or change the port in docker-compose.yml"

  - name: api-server-reachable
    type: http_reachable
    severity: info
    options:
      address: "http://localhost:8080/healthz"
      timeout_ms: "2000"
```
